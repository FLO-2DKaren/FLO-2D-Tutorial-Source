Tailings Dam Model#

Overview

In this tutorial, use QGIS and the tailings dam tool to create a tailings dam hydrograph and simulate a tailings dam failure. The following video covers the Data Assembly Process for a Tailings Dam Study.

Required data

The required data is in Module 7.

The following video will use the tutorial steps below to create the tailings dam hydrograph and run the model.

Step 1: Load the project#

  1. Start with the project from Module 7.

  2. If necessary, load it into QGIS. Open QGIS and drag the Tailings Dam Module 7.qgz file into the project.

  3. Save the project.

  1. Click Yes to load the model.

Step 2. Find inflow node#

  1. Zoom in on the inflow node.

  2. Open the Boundary Condition Editor

  3. Check the Center check box.

  4. Click the Tailings Dam Loc

  1. Zoom in with the mouse wheel. This is the inflow node location where the breach starts.

Step 3. Run the Tailings Dam Tool#

  1. Click the run Tailings Dam Tool button.

  1. Set both file locations to the Module 7 folder.

Step 4. Review Dam geometry#

Dam Height = 86 m

Downstream Slope = 1.5:1

Figure 1. Dam Profile. Source: Feijao Dam 1 Expert Panel Appendices (2019).

Freeboard = 5m

Figure 2. Dam geometry at crest. Source: Periodic Safety Review Vale TUV SUD (2017).

Step 5. Dam and Tailings Volume#

Volume Solids: 7.53X10^6 m3

Actual Volume: 8X10^6 m3

Volume to the crest 8.80X10^6 m3

Figure 3. Volume Curve 6.22. Source: Periodic Safety Review Vale TUV SUD (2017).

Step 6. Dam geotech data#

Dam compacted Unit Weight = 19 KN/m3 = 9279.43 kg/m3

Dam Cohesion = 10 KPa = 1,019.72 kg/m2

Angle of friction = 30 degrees

Figure 4. Adopted Geotechnical Parameters. Source: Periodic Safety Review Vale TUV SUD (2017).

Step 7. Foundation geotechnical data from TUV report#

NSPT silty clay between 5 and 11 strokes

NSPT saprolite horizon between 15 and 40 strokes

Unit weight = 20 KN/m3 = 2039.44 kg/m3

Step 8. Saturated tailings depth#

Dam elevation = 942 m

Water depth = 20 m (max) 5 m (min)

Freeboard 5 m

Figure 5. Overlay of the Interpreted Water Surface and the Cross Section 3-3 Profile. Source: Periodic Safety Review Vale TUV SUD (2017).

Step 9. Tailings dam tool#

  1. Fill the data from the previous pages. Start with Options/Units = Metric

  1. On the next, Choose the Static Mode and set the Reservoir Level to Medium.

  2. Click the Create INFLOW.DAT button.

  1. Set the volume to Vrmax.

  2. Change the event time to 0.50 hours. This variable is highly sensitive to the early hazard zone. The report states that in 10 min 75% of the reservoir was drained.

  3. Set the max concentration to 0.50 or 0.60. This value can be used to test sensitivity of the breach runnout and deposition.

  4. The inflow node is 7872.

  5. Click the graph on the bottom left shown in the blue rectangle.

  1. Change the tab to Sediment Concentration by Volume.

  2. Set the graph to the blue box and Save the INFLOW.DAT file.

  3. Save the INFLOW.DAT file to the Module 7 Export folder.

  1. Save the Tailings Breach 1.xml file.

  1. Close the Tailings Dam Tool.

Step 10. Import the inflow data#

  1. Close the message in QGIS.

  1. Click the Import Components button.

  1. Open the CONT.DAT file.

  1. Uncheck Select All, check Inflow Elements and click OK.

  1. Click ok to close the import message.

  1. Click the Schematized layers to User layers conversion button.

  1. Check the boundary conditions box and click OK and then click ok to close the message.

  1. Click the Schematize button on the Boundary Condition Editor widget and then close both messages.

Step 11. Set the Mud/Debris Data#

  1. Click the Mudflow / Sediment transport button.

  1. Fill the dialog box as shown below and click OK.

  1. Click the Control Variable button.

  1. Select Mud/Debris from the drop down box and click Save.

Step 12. Export and run the model#

  1. Export the FLO-2D Data files. Click OK.

  1. Select the Module 7 Export folder.

  1. The data is ready to run.

  1. Correct the paths and click OK to start the simulation.

  1. Results imported from Mapper.

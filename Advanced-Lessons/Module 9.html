Lesson 3 Advanced - Storm Drain Network#

Overview

This lesson will outline how to construct an advanced storm drain network that is ready for FLO-2D Plugin to process.

Required Data

This lesson starts from the end of the Storm Drain - Shapefile Overview. Please complete that lesson and return here.

File

Content

Point shapefile

Inlets/Junctions, Outfalls, Storage Units

Polyline shapefile

Conduit, Pumps, Orifices, Weirs

*.qgz

Project file

*.gpkg

Geopackage file

*.txt

Rating table file

Step 1: Complete previous lesson#

  1. Finish Storm Drain - Shapefile Overview

  1. The Storm drain overview adds the data and gets the project ready. Return to this step once it is finished.

Step 2: Select component fields from the shapefiles#

  1. Expand the Storm Drain Editor widget and click the Select components from shapefile layer button.

  1. Use the Editor to assign all of the fields from the shapefiles.

  1. Once all features are selected in the drop-down menus, click Assign Selected Inlets/Junctions, Outfalls and Conduits to create the data structures of the Storm Drain Components.

  2. The following message will be displayed. Click OK.

Step 3: Assign the link node connections#

  1. Click the Simulate storm drain checkbox.

  2. Click the Auto-assign link nodes button and select Conduits.

  3. Repeat the step 3.2 for Pumps, Orifices, and Weirs

  1. The success message identifies the number of links that were automatically assigned an Inlet and Outlet node name.

Step 4: Import rating tables#

  1. Click the Import Rating Table Button

  1. Navigate to the I4 table with this path. Select the file and click Open.

C:\Users\Public\Documents\FLO-2D PRO Documentation\Example Projects\QGIS Tutorials\QGIS Lesson 3\Advanced Storm Drain Shapefiles\I4-37-32-26-1.txt

  1. The table was imported and assigned to the correct inlet node.

  2. If an error was detected because one Type 4 inlet didn’t match a selected text file, it would be reported to this file:

Warning

If a Python error pops up because the Rating Table Warnings file can’t be written, simply perform this step again. The path correction is applied the second time.

Step 5: Set up the pump data#

  1. Click the Add pump curve button.

  2. Name the Pump “P1”.

  3. Set the Pump Type to Pump4.

  4. Add a description. (not required)

  1. Apply a rating table.

  1. Pump instructions can be found in the EPA SWMM 5.0 Help Document

C:\ProgramData\Microsoft\Windows\Start Menu\Programs\EPA SWMM 5.0\EPA SWMM 5.0 Help

Step 6: Review the orifice data#

  1. Select Orifces from the Links menu.

  1. Review the table to identify the important details for this system.

  1. The Invert of this system is set by the cistern1, cistern2, and cistern3 features.

  2. The inlet offset defines the depth to the opening.

  3. The orifice has a 6 inch circular opening that allows water to transfer to the other cisterns.

  1. Click close to close the window.

Step 7: Review the weir data#

  1. Select Weirs from the Links menu.

  1. Review the table to identify the important details for this system.

  1. This weir is a control that of the cistern overflow. The water is retained in the system until it fills the tanks to 4.5 ft. At 4.5ft depth, the water flows out of a V-Notch weir into an outfall node with free bubble up discharge back onto the surface retention basin to the East.

Step 8: Add external inflows#

  1. Open the Inlet/Outlet Node editor.

  1. Find the inlet I3-38-32-2 and load it in the Dialog Box.

  2. Check the External inflow checkbox and click the adjacent button.

  1. Click the browse button next to the Time Series field on the Inlet/Junction dialog box.

  1. Fill the table with the following data and click Save.

  1. Click save on the updated Inlet/Junction dialog box.

  1. Click the final Save button on the larger dialog box.

  1. The final message alerts the user that the process completed without error but the schematized button is required to trigger the data table update.

Step 9: Schematize the storm drain components#

  1. Click on Schematize Storm Drain Components in the Storm Drain Editor widget.

  1. Once the storm drain components are schematized, the following dialog will appear. Click OK to close.

  1. The storm drain schematized data layers have been completed. The storm drain components are now part of the schematized layers in the project.

Step 10: Export the project#

  1. Click the Set Control Variable icon.

  1. Verify the data and Click Save.

  1. This is a good point to save project.

  1. Export the data files to the Project Folder in QGIS Lesson 3

C:\Users\Public\Documents\FLO-2D PRO Documentation\Example Projects\QGIS Tutorials\QGIS Lesson 3\QGIS Lesson 3 Export Advanced

  1. Click OK to export the main project data files.

  1. Save the swmm.inp file with the next dialog. Make sure the path is still Lesson 3 Export.

  1. Check the .INP control variables and click OK.

  1. These are the storm drain components that were written to the swmm.inp file.

  1. The final dialog will be displayed. The associated storm drain data files are created when the storm drain switch has been turned ON.

Step 11: Set up the storage unit data#

Note

The plugin cannot make a storage unit yet. This step will be finished in EPA SWMM GUI. The tutorial will be updated when the storage units are added to the plugin.

  1. Open EPA SWMM GUI

  1. Drag the swmm.inp file onto EPA SWMM 5.0 GUI.

  2. Zoom to the LID system.

  1. Right click the cistern1, cistern2, and cistern3 nodes. Convert them into Storage Units.

  1. Double click each storage unit and set up the storage data.

  1. Under the Data Tab, click Storage Curves and the + button to create a curve named storage1.

  1. Save and close the EPA SWMM GUI.

  2. Make a backup of the swmm.inp file in case it is overwritten by a plugin export.

Step 12: Run the simulation#

  1. Click the Run FLO-2D Icon.

  1. Set the Project path and the FLO-2D Engine Path and click OK to start the simulation.

  2. Set the FLO-2D Folder.

C:\program files (x86)\flo-2d pro

  1. Set the Project Folder.

C:\Users\Public\Documents\FLO-2D PRO Documentation\Example Projects\QGIS Tutorials\QGIS Lesson 3\QGIS Lesson 3 Export Advanced

  1. After the storm drain model is complete, review FLO-2D Storm Drain Manual Chapter 6 for more details about reviewing results.
